First, the pet needs Accessibility permission

To float over the screen, the pet needs macOS's Accessibility permission.
It's how the pup follows your cursor and watches the Claude window.

1
On first launch, macOS shows a permission dialog — click "Allow".
2
Missed it? Go to System Settings → Privacy & Security → Accessibility
and toggle Electron ON.
💡 Restart the pet once after toggling the permission.
🪝

Connect Claude Code hooks

This is where the pet earns its keep. Connect these 3 hooks and
the pet will visually surface every Claude signal for you.

Stop
When a response wraps up — 🟢 green
Notification
When macOS notifications fire — 🔵 blue
PermissionRequest
When a permission dialog appears — 🔴 red
⏳ Checking status…
Manual setup (advanced)

Add the snippet below to ~/.claude/settings.json.

.claude/settings.json 
{
  "hooks": {
    "Stop": [{ "hooks": [{ "type": "command",
      "command": "input=$(cat); printf '%s' \"$input\" | curl -sf -X POST http://127.0.0.1:47625/claude-code/stop -H 'content-type: application/json' --max-time 5 -d @- >/dev/null 2>&1; exit 0" }]}],
    "Notification": [{ "hooks": [{ "type": "command",
      "command": "input=$(cat); printf '%s' \"$input\" | curl -sf -X POST http://127.0.0.1:47625/claude-code/notification -H 'content-type: application/json' --max-time 5 -d @- >/dev/null 2>&1; exit 0" }]}],
    "PermissionRequest": [{ "hooks": [{ "type": "command",
      "command": "input=$(cat); printf '%s' \"$input\" | curl -sf -X POST http://127.0.0.1:47625/claude-code/permission-request -H 'content-type: application/json' --max-time 5 -d @- >/dev/null 2>&1; exit 0" }]}]
  }
}
